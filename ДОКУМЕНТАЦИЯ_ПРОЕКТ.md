## 1. Въведение

### 1.1. Цел на документа

Настоящата документация описва разработването и функционирането на информационната система за управление на ресторант "Златен Балкан". Документът съдържа описание на функционалността, архитектурата, инсталацията и използването на системата.

## 2. Описание на проекта

### 2.1. Предназначение

Системата "Златен Балкан" е предназначена за автоматизиране и оптимизиране на процесите в ресторант, като:

- Ускорява обслужването на клиенти
- Намалява грешките при предаване на поръчки
- Осигурява реално време комуникация между отделите
- Осигурява проследяване на финансовите показатели
- Позволява онлайн резервации от клиенти

### 2.2. Основни функции

**За сервитьорите:**
- Преглед на наличните маси
- Създаване на нови поръчки
- Добавяне на артикули от менюто към поръчките
- Проследяване на статуса на поръчките
- Обработка на плащания (в брой и с карта)
- Маркиране на поръчки като обслужени

**За кухнята:**
- Преглед на чакащите поръчки
- Маркиране на поръчки като "в приготвяне"
- Маркиране на поръчки като "готови"
- Статистика за дневната активност

**За мениджърите:**
- Преглед на дневни, седмични и месечни приходи
- Анализ на най-поръчваните артикули
- Финансови отчети с филтриране по дати
- Статистика по методи на плащане

**За клиентите:**
- Онлайн резервация на маси
- Преглед на налични маси по дата и час
- Получаване на потвърждение за резервация

## 3. Функционални изисквания

### 3.1. Управление на менюто

**Описание:**
Системата поддържа пълноценен каталог на ястия и напитки с категоризация и цени.

**Функции:**
- Въвеждане на артикули в менюто с име, описание, цена и категория
- Организиране на артикулите по категории (Предястия, Салати, Супи, Пици, Основни ястия, Десерти, Напитки и др.)
- Маркиране на артикули като налични/неналожни
- Автоматично изчисление на цените при създаване на поръчки

**Категории в менюто:**
1. Предястия
2. Салати
3. Супи
4. Пици
5. Основни ястия
6. Десерти
7. Безалкохолни напитки
8. Сокове
9. Кафе
10. Вина
11. Бира
12. Коктейли
13. Силни напитки

### 3.2. Регистриране на поръчки

**Описание:**
Сервитьорите могат да създават поръчки за различни маси, като избират артикули от менюто.

**Функции:**
- Избор на маса от списъка с налични маси
- Добавяне на артикули от менюто с указание на количество
- Автоматично изчисление на общата стойност на поръчката
- Записване на часа на поръчката
- Добавяне на бележки към поръчката (незадължително)
- Маркиране на масата като заета при създаване на поръчка

**Процес:**
1. Сервитьорът избира маса от списъка
2. Кликва върху бутон "Нова поръчка"
3. Избира артикули от менюто, като кликва върху тях
4. Уточнява количествата
5. Потвърждава поръчката
6. Поръчката се изпраща веднага към кухнята

### 3.3. Известяване на кухнята

**Описание:**
Веднага след създаване на поръчка, тя се появява в кухненския панел за обработка.

**Функции:**
- Автоматично изпращане на новите поръчки към кухнята
- Показване на детайлна информация за всяка поръчка:
  - Номер на масата
  - Име на сервитьора
  - Час на поръчката
  - Списък с всички артикули и количества
  - Текущ статус на поръчката
- Филтриране на поръчки по статус (Чакащи, В приготвяне, Готови)
- Статистика в реално време

### 3.4. Проследяване на статус

**Описание:**
Системата поддържа пълен жизнен цикъл на поръчката с различни статуси.

**Статуси на поръчката:**
1. **Pending (Чакаща)** - Поръчката е създадена и чака обработка от кухнята
2. **Preparing (В приготвяне)** - Кухнята е започнала приготвянето
3. **Ready (Готова)** - Поръчката е готова за сервиране
4. **Served (Обслужена)** - Поръчката е сервирана на клиентите
5. **Paid (Платена)** - Поръчката е платена и масата е освободена

**Функции:**
- Кухнята може да маркира поръчка като "В приготвяне"
- Кухнята може да маркира поръчка като "Готова"
- Сервитьорът може да маркира поръчка като "Обслужена"
- Сервитьорът обработва плащането и поръчката става "Платена"
- Автоматично освобождаване на масата след плащане

### 3.5. Сметка и плащане

**Описание:**
Системата автоматично генерира сметки и поддържа два метода на плащане.

**Функции:**
- Автоматично изчисление на общата стойност на поръчката
- Показване на всички поръчани артикули с цени
- Два метода на плащане:
  - **В брой (Cash)** - Плащане на каса
  - **С карта (Card)** - Плащане чрез Stripe
- Записване на метода на плащане в базата данни
- Автоматично маркиране на поръчката като платена
- Автоматично освобождаване на масата

### 3.6. Резервации на маси

**Описание:**
Клиентите могат да резервират маси онлайн предварително.

**Функции:**
- Публична страница за резервации (достъпна без регистрация)
- Избор на дата и час за резервация
- Указание на брой гости
- Автоматично показване на налични маси
- Проверка за конфликти (предотвратяване на двойно резервиране)
- Проверка на капацитета на масите
- Генериране на потвърждение за резервация

**Логика за проверка:**
- Резервациите са за 2-часов период
- Системата проверява за припокриващи се резервации
- Системата проверява дали масата е заета
- Системата филтрира масите според капацитета

## 4. Технологии и инструменти

### 4.1. Backend технологии

**ASP.NET Core 8.0 MVC**
- Модел-Изглед-Контролер (MVC) архитектура
- Razor views за динамично генериране на HTML
- Dependency Injection за управление на зависимости
- Middleware за автентикация и авторизация

**Entity Framework Core 8.0**
- ORM (Object-Relational Mapping) за работа с базата данни
- Code First подход за моделиране на данните
- Миграции за управление на схемата на базата данни
- LINQ заявки за достъп до данни

**C# (.NET 8.0)**
- Обектно-ориентиран език за програмиране
- Асинхронно програмиране (async/await)
- LINQ за работа с данни

### 4.2. Frontend технологии

**HTML5 и CSS3**
- Семантичен HTML

## 5. Архитектура на системата

### 5.1. Обща архитектура

Системата следва MVC (Model-View-Controller) архитектурен модел:

```
┌─────────────────────────────────────────────────┐
│                  КЛИЕНТ                         │
│         (Браузър - HTML/CSS/JavaScript)        │
└──────────────────┬──────────────────────────────┘
                   │
                   │ HTTP Request/Response
                   │
┌──────────────────▼──────────────────────────────┐
│              CONTROLLER LAYER                   │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐    │
│  │  Waiter  │  │ Kitchen  │  │ Manager  │    │
│  │Controller│  │Controller│  │Controller│    │
│  └──────────┘  └──────────┘  └──────────┘    │
└──────────────────┬──────────────────────────────┘
                   │
                   │ Business Logic
                   │
┌──────────────────▼──────────────────────────────┐
│              SERVICE LAYER                      │
│  ┌──────────┐  ┌──────────────┐                │
│  │  Order   │  │ Reservation  │                │
│  │ Service  │  │   Service    │                │
│  └──────────┘  └──────────────┘                │
└──────────────────┬──────────────────────────────┘
                   │
                   │ Data Access
                   │
┌──────────────────▼──────────────────────────────┐
│              DATA LAYER                         │
│     Entity Framework Core + SQL Server          │
└─────────────────────────────────────────────────┘
```

### 5.2. Компоненти на системата

**Controllers (Контролери):**
- `HomeController` - Начална страница
- `AuthController` - Автентикация и авторизация
- `WaiterController` - Функционалност за сервитьори
- `KitchenController` - Функционалност за кухня
- `ManagerController` - Отчети и статистика
- `ReservationController` - Резервации
- `SetupController` - Създаване на мениджър акаунт

**Services (Услуги):**
- `OrderService` - Бизнес логика за поръчки
- `ReservationService` - Бизнес логика за резервации

**Models (Модели):**
- `User` - Потребители (сервитьори, кухня, мениджъри)
- `Table` - Маси в ресторанта
- `MenuItem` - Артикули от менюто
- `Order` - Поръчки
- `OrderItem` - Артикули в поръчка
- `Reservation` - Резервации

**Views (Изгледи):**
- Razor views за всеки контролер
- Shared layouts и partial views
- Responsive дизайн

### 5.3. Поток на данни

**Създаване на поръчка:**
1. Сервитьорът избира маса и артикули (Frontend)
2. JavaScript изпраща AJAX заявка (HTTP Request)
3. WaiterController получава заявката
4. OrderService обработва бизнес логиката
5. ApplicationDbContext записва данните в базата
6. Отговор се връща към браузъра (HTTP Response)
7. Поръчката се появява в кухненския панел

**Промяна на статус:**
1. Кухнята кликва "Започни приготвяне"
2. AJAX заявка към KitchenController
3. OrderService обновява статуса
4. Базата данни се актуализира
5. Сервитьорът вижда промяната при следващото опресняване

- Много резервации към една маса (M:1)

### 9.3. Автентикация и авторизация

**Cookie-based Authentication:**
- Използва се ASP.NET Core Cookie Authentication
- Сесиите са валидни за 8 часа
- Ролите се контролират чрез `[Authorize]` атрибути

**Роли:**
- `Waiter` - Доступ до сервитьорски панел
- `Kitchen` - Доступ до кухненски панел
- `Manager` - Доступ до всички панели + отчети

**Защита на пароли:**
- BCrypt алгоритъм за хеширане
- Паролите никога не се съхраняват като plain text

- `POST /Waiter/ProcessCashPayment` - Обработка на плащане в брой

- `POST /Waiter/ConfirmCardPayment` - Потвърждение на плащане с карта

**Kitchen Endpoints:**
- `GET /Kitchen` - Кухненски панел

- `POST /Kitchen/MarkItemReady` - Маркиране на артикул като готов

**Manager Endpoints:**
- `GET /Manager` - Мениджърски dashboard
- `GET /Manager/RevenueReport` - Финансов отчет
- `GET /Manager/PopularItems` - Анализ на популярни артикули

**Reservation Endpoints:**
- `GET /Reservation/Book` - Форма за резервация

**Auth Endpoints:**
- `GET /Auth/Login` - Страница за вход

**Защита срещу SQL Injection:**
- Entity Framework Core използва parameterized queries
- Всички входни данни се валидират

**Защита срещу XSS:**
- Razor views автоматично encode HTML
- Валидация на всички потребителски входове

**Защита на пароли:**
- BCrypt хеширане
- Минимална дължина на парола: 6 символа

**Авторизация:**
- Role-based access control
- Защита на endpoints чрез `[Authorize]` атрибути

- Поръчката се появява в кухнята
- Масата става заета

**Очакван резултат:**
- Статусът се променя на "В приготвяне"
- След това на "Готова"
- Сервитьорът вижда промяната

#### Сценарий 3: Обработка на плащане

**Предусловия:**
- Поръчката е със статус "Обслужена"

**Стъпки:**
1. Кликнете на "Плащане"
2. Изберете "Плащане в брой"
3. Потвърдете

**Очакван резултат:**
- Поръчката става "Платена"
- Масата се освобождава
- Поръчката изчезва от активните

#### Сценарий 4: Резервация

**Предусловия:**
- Няма резервации за избраното време

**Стъпки:**
1. Изберете дата и час
2. Въведете брой гости
3. Проверете наличност
4. Изберете маса
5. Потвърдете резервацията

**Очакван резултат:**
- Резервацията се създава успешно
- Показва се страница за потвърждение
- Масата не може да бъде резервирана отново за същото време

### 10.2. Гранични случаи

**Тест 1: Поръчка без артикули**
- Очаквано поведение: Валидационна грешка

**Тест 2: Резервация за минало време**
- Очаквано поведение: Валидационна грешка

**Тест 3: Резервация за вече заета маса**
- Очаквано поведение: Масата не се показва като налична

**Тест 4: Плащане на вече платена поръчка**
- Очаквано поведение: Поръчката не може да бъде платена повторно

### 10.3. Производителност

**Тестове за производителност:**
- Време за зареждане на сервитьорския панел: < 2 секунди
- Време за създаване на поръчка: < 1 секунда
- Време за обновяване на статус: < 0.5 секунди
- Автоматично опресняване: На всеки 30 секунди

### 11.3. Кодови примери

#### Пример 1: Създаване на поръчка

```csharp
public async Task<Guid> CreateOrderAsync(Guid tableId, Guid waiterId, List<OrderItemRequest> items)
{
    var order = new Order
    {
        Id = Guid.NewGuid(),
        TableId = tableId,
        WaiterId = waiterId,
        Status = OrderStatus.Pending,
        CreatedAt = DateTime.UtcNow,
        UpdatedAt = DateTime.UtcNow
    };

    decimal totalAmount = 0;
    foreach (var item in items)
    {
        var menuItem = await _context.MenuItems.FindAsync(item.MenuItemId);
        if (menuItem != null)
        {
            var orderItem = new OrderItem
            {
                Id = Guid.NewGuid(),
                OrderId = order.Id,
                MenuItemId = item.MenuItemId,
                Quantity = item.Quantity,
                Price = menuItem.Price
            };
            order.OrderItems.Add(orderItem);
            totalAmount += menuItem.Price * item.Quantity;
        }
    }

    order.TotalAmount = totalAmount;
    
    var table = await _context.Tables.FindAsync(tableId);
    if (table != null)
    {
        table.IsOccupied = true;
    }

    _context.Orders.Add(order);
    await _context.SaveChangesAsync();

    return order.Id;
}
```

#### Пример 2: Проверка на наличност

```csharp
public async Task<bool> IsTableAvailableAsync(Guid tableId, DateTime reservationDateTime)
{
    var table = await _context.Tables.FindAsync(tableId);
    if (table == null || table.IsOccupied)
    {
        return false;
    }

    var reservationStart = reservationDateTime;
    var reservationEnd = reservationDateTime.AddHours(2);

    var conflictingReservations = await _context.Reservations
        .Where(r => r.TableId == tableId)
        .Where(r => r.ReservationDateTime < reservationEnd && 
                   r.ReservationDateTime.AddHours(2) > reservationStart)
        .CountAsync();

    return conflictingReservations == 0;
}
```
